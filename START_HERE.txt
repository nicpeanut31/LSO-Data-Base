WHAT THIS ONLINE VERSION DOES
-----------------------------
- Stores one shared copy of members, events, attendance, settings, and activity.
- Synchronizes changes across approved devices approximately every 5 seconds.
- Saves changes immediately to a local cache and queues them if the connection
  briefly drops.
- Uses username and password only. Email remains optional contact information.
- Keeps new registrations Pending until an Administrator approves them.
- Does not use Supabase email authentication and does not send confirmation mail.

MOVING EXISTING LOCAL RECORDS ONLINE
------------------------------------
Use the SAME browser, device, and exact GitHub Pages address where the current
local records are visible. Do not clear browser data first.

When the default Administrator logs in and the shared database is empty, the
website automatically copies the existing members, events, attendance, settings,
and activity records into Supabase. Other devices will then load the same data.

SECURITY NOTES
--------------
- The publishable key may be stored in the public website configuration.
- Never place a secret key or service_role key in GitHub.
- Direct access to the database tables is blocked.
- Passwords are stored as bcrypt hashes in PostgreSQL.
- Browser sessions use random private tokens and expire after 30 days.

TROUBLESHOOTING
---------------
"Required database functions are missing"
Run the latest supabase-setup.sql completely.

"Supabase project could not be reached"
Copy the exact Project URL from Supabase > Connect and confirm the project is
active, not paused or deleted.

"Invalid username or password"
Confirm the SQL ran successfully. The default account is created automatically
when the website connects for the first time.
